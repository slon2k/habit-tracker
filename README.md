# Habit Tracker API

A backend demo project focused on building APIs with production-minded engineering practices.

## Overview

This repository showcases how to design and ship a clean ASP.NET Core API with:

- clear project structure
- strong code quality standards
- built-in observability
- containerized local development
- CI validation on every change

## Tech Stack

- .NET 10 / ASP.NET Core
- OpenTelemetry
- Docker / Docker Compose
- GitHub Actions

## Run Locally

1. Restore and build the solution
2. Run the API project
3. Open the health endpoint at /health

You can also run the full local stack with Docker Compose.

## Project Status

This is a training project, not a production system.
Some features are intentionally simplified to keep focus on architecture, quality, and delivery workflow.
